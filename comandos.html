Fixando os comandos GIT e GITHUB

Sempre iniciar uma pasta com o comando GIT INIT


Configuando o GIT

GIT CONFIG -- GLOBAL USER.NAME - para configurar o usuario

GIT CONFIG -- GLOBAL USER.EMAIL - para configurar o e-mail da conta GIT


Para verificar da conta configurada é só usar os comandos:

GIT CONFIG USER.NAME e GIT CONFIG USER.EMAIL

Comandos

PWD - Mostra onde você está.

CD - Comando para trocar de pasta. ex: cd /c

CLEAR - Comando para limpar o terminal.

LS - Comando para mostrar todos os arquivos na pasta.

MKDIR - Comando para criar uma nova pasta

TOUCH - Comando para criar um arquivo.

------------------------------------------------------------------------------------------------------------------------------------------

Comando Basicos

Abrir uma pasta e depois ir em GIT BASH HERE para abrir o terminal do GIT. Depois iniciar o projeto com o comando GIT INIT

para exibir arquivos ocultos é só o comando LS -A

GIT ADD - Comando para adicionar o arquivo ao container.

GIT ADD . ou * - Comando pra adicionar tudo de uma vez.

GIT STATUS - Comando para verificar o que esta dentro do container.

GIT COMMIT -M - Comando para identificar o e armazenar/subir o arquivo que está no container.

OBS> GIT COMMIT -AM "Sobe tudo de uma vez" sobe tudo de uma vez

Ex: TOUCH pra criar arquivo a.txt. GIT STATUS pra verificar. GIT ADD para adicionar. Depois de adicionar e identificar é só comitar com o GIT COMMIT -M "Subindo arquivo".

GIT LOG - Comando para mostrar o log.

GIT LOG --ONELINE - Comando para mostrar abreviar.

------------------------------------------------------------------------------------------------------------------------------------------

Controle de Versão

Rastreando e recuperando versões anteriores do projeto.

GIT LOG --ONELINE - Comando pra mostrar o numero do HASH. São sempre os 7 primeiros.

HASH = identificador de cada commit.

GIT LOG --GRAPH - Comando pra mostrar o desenho de GRAPH.

GIT LOG --ONELINE --GRAPH - Comando pra abreviar o GRAPH.

GIT BRANCH - Comando usado verificar em qual RAMO você esta ex: main, master ou outro.

Aprendendo a rastrear as mudanças no projeto e recuperar uma versão anterior.

Para recuperar uma versão antetior temos que apontar o HEAD para o COMMIT / HASH que desejamos acessar.

GIT CHECKOUT - Comando para trocar de HASH, basta colocar o identifcador de 7 digitos que ele volta pro HASH escolhido. EX: GIT CHECKOUT 807b130

Para voltar é só digitar GIT CHECKOUT e o nome do HASH

Desfazendo Alterações e Excluindo um Commit

GIT DIFF - Comando para mostrar as alterações que foram feitas no arquivo.

para desfazer alguma alteração basta seguir as instruções do GIT STATUS

GIT CHECKOUT + o nome do arquivo. Apos o comando ele vai descartar as alterações

GIT RESET --HARD - Comando para excluir o COMMIT. Bastar usar o comando e colocar o penultimo HASH.
